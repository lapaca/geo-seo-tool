# GEO+SEO 自动优化工具 - 产品需求文档

## 1. 产品概述

### 1.1 产品定位
面向个人站长的一站式 GEO（生成式引擎优化）+ SEO（搜索引擎优化）分析与优化工具。用户输入网页 URL，系统自动抓取并分析页面内容，给出 SEO/GEO 问题诊断和优化建议，并通过 AI 生成优化后的内容草稿，用户确认后可导出使用。

### 1.2 核心价值
- **降低门槛**：个人站长无需 SEO/GEO 专业知识，输入 URL 即可获得专业级优化建议
- **安全可控**：不直接改写原站内容，采用「分析 → 建议 → 对比预览 → 确认导出」模式
- **AI 驱动**：使用 Claude API 进行内容分析和优化建议生成，质量高于纯规则引擎

### 1.3 目标用户
个人站长、博主，有内容优化需求但缺乏 SEO/GEO 专业知识的用户。

---

## 2. 核心流程

```
用户输入 URL
    ↓
系统抓取页面内容（HTML、Meta、结构化数据等）
    ↓
规则引擎 + AI 分析（SEO 检查 + GEO 检查）
    ↓
生成诊断报告（问题清单 + 评分）
    ↓
AI 生成优化建议和改写草稿
    ↓
用户对比预览（原文 vs 优化版）
    ↓
用户逐条确认/编辑
    ↓
导出优化后的内容
```

## 3. 功能模块

### 3.1 用户系统

#### 3.1.1 注册/登录
- 支持邮箱 + 密码注册登录
- 支持 GitHub OAuth 登录（可选）
- JWT Token 认证

#### 3.1.2 用户仪表盘
- 查看历史分析记录
- 查看已保存的优化建议

### 3.2 页面抓取模块

#### 3.2.1 输入
- 用户输入单个页面 URL
- URL 格式校验

#### 3.2.2 抓取内容
- HTML 全文
- `<title>` 标签
- `<meta description>` 标签
- `<meta keywords>` 标签
- Open Graph 标签（og:title, og:description, og:image）
- 结构化数据（JSON-LD / Schema.org）
- 标题层级（H1-H6）
- 图片 alt 属性
- 内链/外链
- 页面加载速度（基础指标）
- robots.txt / sitemap 检测
- canonical 标签
- 移动端适配 meta viewport

### 3.3 SEO 分析模块

#### 3.3.1 检查项（规则引擎）

| 检查项 | 规则 | 严重度 |
|--------|------|--------|
| Title 标签 | 存在且长度 30-60 字符 | 高 |
| Meta Description | 存在且长度 120-160 字符 | 高 |
| H1 标签 | 唯一且存在 | 高 |
| 标题层级 | H1→H2→H3 结构合理 | 中 |
| 图片 Alt | 所有图片都有 alt 属性 | 中 |
| 内链数量 | 至少有 3 个内链 | 低 |
| 外链数量 | 适量外链且无死链 | 低 |
| HTTPS | 是否使用 HTTPS | 高 |
| Canonical | 存在 canonical 标签 | 中 |
| Open Graph | OG 标签完整 | 中 |
| 移动端适配 | 有 viewport meta | 中 |
| 页面速度 | HTML 大小合理（< 3MB） | 中 |
| robots.txt | 可访问且未阻止爬虫 | 高 |
| sitemap | 存在 sitemap.xml | 低 |

#### 3.3.2 SEO 评分
- 根据检查项通过/失败情况，按权重计算综合评分（0-100）
- 分级：优秀（80+）/ 良好（60-79）/ 需优化（40-59）/ 较差（<40）

### 3.4 GEO 分析模块

#### 3.4.1 检查项

| 检查项 | 说明 | 严重度 |
|--------|------|--------|
| 结构化数据 | 是否有 JSON-LD/Schema.org 标记 | 高 |
| 内容清晰度 | 段落是否有明确的主题句，AI 能否提取核心观点 | 高 |
| 引用权威性 | 是否引用可信来源、数据、统计 | 高 |
| 事实密度 | 内容中具体事实/数据/数字的密度 | 中 |
| 问答格式 | 是否包含 FAQ 或问答结构（利于 AI 引用） | 中 |
| 列表/表格 | 是否使用结构化列表和表格（利于 AI 解析） | 中 |
| 内容唯一性 | 内容是否有独特观点（非纯搬运） | 高 |
| 语义标记 | 是否正确使用 HTML5 语义标签（article, section, nav 等） | 中 |
| 摘要/TL;DR | 是否有文章摘要或总结 | 低 |

#### 3.4.2 GEO 评分
- 综合评分 0-100
- 分级同 SEO 评分

### 3.5 AI 优化建议模块

#### 3.5.1 优化内容生成（Claude API）
针对诊断出的每个问题，AI 生成具体优化建议：

- **Title 优化**：生成 2-3 个优化后的 title 候选
- **Meta Description 优化**：生成优化后的 description
- **H1 优化**：生成优化后的 H1
- **内容改写建议**：针对 GEO 问题，建议如何增加引用、结构化内容、添加摘要等
- **结构化数据生成**：自动生成 JSON-LD 代码片段
- **Alt 文本生成**：为缺少 alt 的图片生成描述
- **FAQ Schema 生成**：基于内容生成 FAQ 结构化数据

#### 3.5.2 对比预览
- 左右分栏对比：原始内容 vs AI 优化版
- 差异高亮显示
- 用户可逐条接受/拒绝/编辑每个优化建议

### 3.6 导出模块

#### 3.6.1 导出格式
- HTML 代码片段（可直接粘贴到网站）
- JSON 格式（结构化数据���
- Markdown 格式（内容部分）
- 一键复制到剪贴板

## 4. 页面设计

### 4.1 页面列表

| 页面 | 路径 | 说明 |
|------|------|------|
| 首页/Landing | `/` | 产品介绍 + URL 输入框 |
| 登录页 | `/login` | 邮箱密码登录 |
| 注册页 | `/register` | 邮箱密码注册 |
| 分析结果页 | `/report/[id]` | 分析报告 + 优化建议 |
| 仪表盘 | `/dashboard` | 历史记录列表 |

### 4.2 首页
- 顶部导航栏（Logo、登录/注册、仪表盘）
- 中间大标题 + 简介
- URL 输入框 + "开始分析" 按钮
- 底部展示功能特点

### 4.3 分析结果页（核心页面）
- **顶部概览**：URL、分析时间、SEO 评分、GEO 评分（双圆环图）
- **问题清单 Tab**：按严重度排列的问题列表，每项可展开查看详情
- **优化建议 Tab**：AI 生成的优化建议，左右对比预览
- **导出按钮**：导出选中的优化内容

## 5. 技术架构

### 5.1 技术栈
- **框架**：Next.js 14+ (App Router)
- **语言**：TypeScript
- **样式**：Tailwind CSS + shadcn/ui
- **数据库**：SQLite (Prisma ORM)，MVP 阶段够用
- **认证**：NextAuth.js（邮箱密码 + 可选 OAuth）
- **AI**：Claude API (Anthropic SDK)
- **网页抓取**：cheerio（HTML 解析）+ fetch（抓取）
- **部署**：Vercel

### 5.2 API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/register` | 用户注册 |
| POST | `/api/auth/login` | 用户登录 |
| POST | `/api/analyze` | 提交 URL 开始分析 |
| GET | `/api/report/[id]` | 获取分析报告 |
| GET | `/api/reports` | 获取历史报告列表 |
| POST | `/api/optimize` | 请求 AI 生成优化建议 |
| POST | `/api/export` | 导出优化内容 |

### 5.3 数据模型

```
User
├── id
├── email
├── password (hashed)
├── name
├── createdAt

Report
├── id
├── userId (FK → User)
├── url
├── seoScore
├── geoScore
├── crawlData (JSON) — 抓取的原始数据
├── seoIssues (JSON) — SEO 问题列表
├── geoIssues (JSON) — GEO 问题列表
├── optimizations (JSON) — AI 优化建议
├── status (pending / analyzing / completed / failed)
├── createdAt
```

## 7. 非功能需求

- **性能**：单页面分析完成时间 < 30s（含 AI 生成）
- **安全**：密码 bcrypt 加密，API 限流防滥用
- **可用性**：响应式设计，移动端可用
- **数据**：用户数据本地 SQLite 存储，不依赖外部数据库服务
